### Configuration

See more on the [official website](https://swiperjs.com/element#updating-parameters)

```astro
<!-- With background -->
<HeroSlider
  items={[...]}
>
  <!-- Background container -->
  <Fragment slot="bg">
      <div class="absolute inset-0 bg-primary/10 dark:bg-card"></div>
    </Fragment>
</HeroSlider>

<!-- With custom props -->
<HeroSlider
  classes={{ 
    height: 'h-[50rem]',
    container: 'py-4',
    loading: 'bg-card',
  }}
  id='heroSection'
  withPagination={true}
  withNavigation={true}
  autoplay={true}
  isDark="true"
  items={[...]}
/>
```

## Props

### HeroSlider

<div className="overflow-x-auto">
  <div className="min-w-xl">
    | Prop | Type | Default | Description |
    |------------|------------|------------|------------|
    | `height` | `string` | `h-[28rem]` | Set the height of the slider and its layers |
    | `classes.slider` | `string` | - | CSS classes for the `swiper-container` container |
    | `classes.container` | `string` | - | CSS classes for the `WidgetWrapper` container |
    | `classes.loading` | `string` | - | CSS classes for loading layout |
    | `id` | `string` | - | |
    | `withPagination` | `boolean` | `true` | Pagination visibility |
    | `withNavigation` | `boolean` | `true` | Navigation visibility |
    | `autoplay` | boolean | `true` | |
    | `isDark` | boolean | `false` | |
    | `items` | `HeroSlide[]` | `[]` | Array of `<HeroSlide />` components |
  </div>
</div>
